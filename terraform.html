Terraform核心优势

声明式配置

使用HCL语言描述基础设施状态，无需关心实现细节，自动计算资源依赖关系。

多云编排

状态管理

记录基础设施状态变化，支持版本控制和团队协作，避免配置漂移。

Terraform标准工作流程

编写配置

使用HCL语言定义资源需求，模块化组织代码结构，支持变量和输出。

初始化

执行terraform init下载所需provider插件，准备后端存储。

计划预览

运行terraform plan生成执行计划，确认变更内容无误。

应用变更

执行terraform apply创建或更新基础设施，自动处理依赖关系。

销毁资源

通过terraform destroy清理不再需要的资源，避免费用浪费。

Terraform配置示例

main.tf 
# 配置AWS提供商
provider "aws" {
  region = "us-west-2"
}

# 创建VPC网络
resource "aws_vpc" "main" {
  cidr_block = "10.0.0.0/16"
  
  tags = {
    Name = "Production-VPC"
  }
}

# 创建公共子网
resource "aws_subnet" "public" {
  vpc_id     = aws_vpc.main.id
  cidr_block = "10.0.1.0/24"
  
  tags = {
    Name = "Public-Subnet"
  }
}

# 创建安全组
resource "aws_security_group" "web" {
  name        = "web-traffic"
  description = "Allow HTTP/HTTPS traffic"
  vpc_id      = aws_vpc.main.id

  ingress {
    from_port   = 80
    to_port     = 80
    protocol    = "tcp"
    cidr_blocks = ["0.0.0.0/0"]
  }

  egress {
    from_port   = 0
    to_port     = 0
    protocol    = "-1"
    cidr_blocks = ["0.0.0.0/0"]
  }
}

# 输出VPC ID
output "vpc_id" {
  value = aws_vpc.main.id
}
