为什么选择Ansible?

无代理架构

通过SSH协议直接管理节点，无需在目标主机安装任何客户端代理，简化部署和维护。

YAML语法

使用易读的YAML格式编写Playbook，降低学习曲线，便于版本控制和团队协作。

模块化设计

丰富的内置模块支持各种运维任务，从软件安装到云资源管理，覆盖全场景需求。

Ansible架构原理

控制节点

运行Ansible的主机，包含Inventory文件和Playbook，通过SSH管理目标节点。

被管节点

无需安装特殊软件，只需支持Python和SSH即可被Ansible管理。

模块系统

执行特定任务的代码单元，通过临时命令或Playbook调用，结果以JSON格式返回。

插件系统

扩展Ansible核心功能，包括连接插件、过滤器插件、回调插件等。

Playbook示例

webserver.yml 
---
# 安装并配置Apache Web服务器
- hosts: webservers
  become: yes
  vars:
    http_port: 80
    max_clients: 200

  tasks:
    - name: 安装Apache
      yum:
        name: httpd
        state: latest

    - name: 写入配置文件
      template:
        src: templates/httpd.conf.j2
        dest: /etc/httpd/conf/httpd.conf
      notify:
        - 重启Apache

    - name: 确保Apache正在运行
      service:
        name: httpd
        state: started
        enabled: yes

  handlers:
    - name: 重启Apache
      service:
        name: httpd
        state: restarted

Playbook结构

YAML格式，包含hosts、tasks、handlers等部分

幂等性

多次执行结果一致，避免重复操作

模块调用

使用yum、template、service等内置模块

快速开始

安装Ansible

# 在控制节点上安装

$ sudo apt update

$ sudo apt install ansible -y

支持多种平台安装：

RHEL/CentOS Ubuntu/Debian macOS Windows(WSL)

基本使用

测试连接

$ ansible all -m ping

执行临时命令

$ ansible webservers -a "uptime"

运行Playbook

$ ansible-playbook site.yml
