Kubernetes核心优势

自动化部署

自动部署容器化应用，实现滚动更新和回滚，确保零停机部署。

弹性伸缩

根据CPU使用率或自定义指标自动扩展或缩减应用实例数量。

自愈能力

自动重启失败的容器，替换不可用的节点，保持应用高可用性。

Kubernetes架构原理

控制平面

包含API Server、Scheduler、Controller Manager等组件，负责集群管理和决策。

工作节点

运行容器化应用，包含kubelet、kube-proxy和容器运行时等组件。

etcd

分布式键值存储，保存集群所有配置数据和状态信息。

网络模型

每个Pod获得唯一IP地址，Pod间可直接通信，无需NAT。

Kubernetes常用命令

Terminal 
# 查看集群信息
kubectl cluster-info

# 部署应用
kubectl create deployment nginx --image=nginx

# 查看Pod状态
kubectl get pods

# 查看服务
kubectl get services

# 扩展部署
kubectl scale deployment nginx --replicas=3

# 查看日志
kubectl logs [pod-name]

# 进入容器
kubectl exec -it [pod-name] -- /bin/bash

# 删除资源
kubectl delete deployment nginx

快速开始使用Kubernetes

安装Minikube

# MacOS使用Homebrew安装

$ brew install minikube

# 启动本地集群

$ minikube start

# 验证安装

$ kubectl version

其他安装方式：

Windows Linux Docker

部署第一个应用

创建部署

$ kubectl create deployment nginx --image=nginx

暴露服务

$ kubectl expose deployment nginx --port=80

访问应用

$ minikube service nginx
