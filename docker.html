Docker容器网络深度解析

Docker容器网络是容器化技术中最复杂也是最重要的部分之一。理解Docker网络模型和工作原理，能够帮助您构建更安全、更高效的容器化应用。

Docker网络的核心价值：为容器提供隔离的网络环境，同时保持必要的通信能力。

Docker网络基础概念

  • 网络命名空间(Network Namespace)：Linux内核提供的网络隔离机制
  • 虚拟以太网设备(veth pair)：连接不同网络命名空间的虚拟设备
  • 网桥(Bridge)：软件实现的二层网络设备，用于连接多个网络接口
  • iptables/nftables：实现NAT和网络流量控制

Docker网络模型

桥接网络(Bridge)

Docker默认网络模式，通过docker0网桥连接容器。

# 创建自定义桥接网络
docker network create my-bridge
  • 容器间通过IP或容器名通信
  • 支持端口映射到宿主机
  • 默认提供DNS解析服务

Host网络

容器直接使用宿主机的网络栈。

# 使用host网络模式运行容器
docker run --network=host nginx
  • 性能最好，无NAT开销
  • 网络隔离性最差
  • 适合高性能网络应用

覆盖网络(Overlay)

跨主机的容器网络解决方案。

# 创建覆盖网络
docker network create -d overlay my-overlay
  • 支持多主机容器通信
  • 内置加密和负载均衡
  • Swarm集群默认使用

Docker网络模型对比

驱动类型 描述 适用场景
bridge 默认驱动，创建私有内部网络 单主机容器通信
host 直接使用主机网络栈 高性能网络需求
overlay 跨主机容器网络 Swarm集群

网络驱动选择建议

  • 开发测试环境推荐使用bridge驱动，简单易用
  • 生产环境高吞吐量应用考虑host驱动减少开销
  • 跨主机通信必须使用overlay

Docker网络实践指南

1. 自定义网络配置

Docker允许创建自定义网络，提供更灵活的容器连接方式。

# 创建自定义桥接网络，指定子网和网关
docker network create \
  --driver=bridge \
  --subnet=172.28.0.0/16 \
  --gateway=172.28.0.1 \
  my-custom-net

# 使用自定义网络运行容器
docker run -d --name=web \
  --network=my-custom-net \
  nginx:alpine

最佳实践：

  • 为不同应用创建独立的网络，实现网络隔离
  • 使用DNS名称而非IP地址进行容器间通信
  • 为生产环境配置适当的网络驱动和选项

2. 多容器应用网络配置

典型的多容器应用(如Web+DB)网络配置示例。

客户端

Web容器

DB容器

# 创建应用专用网络
docker network create app-network

# 启动数据库容器
docker run -d --name db \
  --network app-network \
  -e POSTGRES_PASSWORD=secret \
  postgres:13

# 启动Web应用容器
docker run -d --name web \
  --network app-network \
  -p 8080:80 \
  -e DB_HOST=db \
  my-web-app

关键点：

  • 所有相关容器加入同一自定义网络
  • 通过容器名称而非IP地址访问服务
  • 仅暴露必要的端口到宿主机

Docker网络故障排查

常见网络问题排查工具

1. 容器网络检查

# 检查容器网络配置
docker inspect [容器名] | grep IPAddress

# 查看容器网络连接
docker network inspect [网络名]

2. 网络连通性测试

# 进入容器测试网络
docker exec -it [容器名] sh

# 测试DNS解析
nslookup google.com

# 测试端口连通性
telnet [目标IP] [端口]

常见问题及解决方案

问题1：容器无法访问外部网络

解决方案：检查宿主机的iptables/nftables规则，确保NAT规则正确配置

问题2：容器间无法通过名称通信

解决方案：确保所有容器在同一自定义网络中，且Docker守护进程的DNS服务正常运行

问题3：端口映射不生效

解决方案：检查端口是否被占用，确保-p参数格式正确(宿主机端口:容器端口)，验证防火墙设置
